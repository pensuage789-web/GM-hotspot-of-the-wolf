数据维护方案

仍然保留最稳妥的做法：用一个独立的 JSON 文件维护所有区域信息，便于你自己随时改。

方案一：继续编辑 macro_data.json（最推荐）

优点是轻量、稳定、无需数据库、无需后端、用记事本或 VS Code 就能改，刷新网页立即生效。

方案二：后续升级为 Google Sheets / Airtable

如果未来你要多人协作维护数据，可以把数据源迁移到表格型数据库，再让网页自动读取。

方案三：正式上线时接入后端

如果以后要做成真正长期在线使用的网站，再接一个轻量 CMS 或后端接口会更规范。
